üîå Networking Options

  • USB-C (Virtual Ethernet): Use USB-C adapters, configure static IPs (e.g., 10.0.0.X).
  • Cat6 Ethernet: Connect via switch or router; drop all outbound traffic using iptables.

üîê Security Isolation

  • Each node can be wiped or replaced.
  • Compromised nodes can't affect others.
  • Nodes can report to a dedicated security monitor (Watchdog Node).

  • Controller Node connects to monitor, keyboard, mouse.

  • Consider using tmux or screen for session multiplexing.

üîÅ Synchronization

  • Use rsync over SSH for syncing logs or memory JSON files.

  • Define envelope files for context transfer between nodes.

üß© Part Salvaging Guide

When salvaging parts from old laptops/desktops:

  • CPU: Only usable if socket-compatible with target motherboard.
  • RAM: Match DDR type (DDR2/3/4), voltage, and slots.
  • Wi-Fi/Bluetooth Cards: Can be removed to eliminate attack surface.
  • Motherboard: Reuse for less critical nodes (Meta, Memory).
  • Drives (HDD/SSD): Use for logging or archive purposes.
  • Power Supplies: Check for consistent voltage/amperage.
  • Displays: Reuse LCDs for controller node or monitoring.
  • Fans/Heatsinks: Useful for passive node cooling.

üéØ Final Tips

  • Start with two nodes: Controller + Inference.
  • Expand to include Memory, Meta, and Watchdog as needed.
  • Document each node with physical labels and a boot guide.
