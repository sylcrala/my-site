Comprehensive BIOS/UEFI Hardening Checklists

General Recommendations (Applicable to All Systems)

  • Set a strong BIOS/UEFI admin password.
  • Disable unused boot devices (e.g. USB, PXE).
  • Enable Secure Boot (after installing necessary keys if using Linux).
  • Disable Legacy Boot / CSM (Compatibility Support Module) if your OS supports UEFI fully.
  • Enable TPM if available (for disk encryption and secure key storage).
  • Enable Intel TXT or AMD SVM/SME for trusted execution if supported.
  • Keep BIOS/UEFI firmware up to date (from manufacturerâ€™s official tools).

Linux-Specific BIOS Settings

  • Legacy USB Support: Leave enabled if you need to use a keyboard in early boot or pre-boot environments. Disable otherwise.
  • XHCI Hand-off: Usually enabled for modern Linux systems. Disable only if facing USB stability issues.
  • UEFI Network Stack: Disable unless you use PXE booting over the network.
  • Virtualization: Enable VT-x/AMD-V and VT-d/IOMMU for advanced container/VM use.
  • Disable "Fast Boot" for better hardware initialization diagnostics.

Windows-Specific BIOS Settings

  • Secure Boot: Enable to ensure only signed OS components are loaded.
  • TPM: Required for BitLocker and Windows Hello security features.
  • Virtualization: Enable Hyper-V, Memory Integrity, Credential Guard with VT-x/AMD-V and VT-d/IOMMU.
  • UEFI Boot: Required for Device Guard and modern security features.
  • Enable BIOS-level anti-rollback or "firmware update protection" if available.

Extra Tips

  • Document current BIOS settings before making changes.
  • Disable Thunderbolt, FireWire, and other high-speed external ports if unused (attack surface reduction).
  • Consider enabling BIOS-level disk encryption if available (e.g., OPAL SSDs).
